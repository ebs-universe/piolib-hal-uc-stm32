29 /*
30  * This implementation is designed for multi-channel ADC sampling with
31  * relatively uniform conversion parameters. Specifically, all enabled
32  * channels are added into a single conversion group per ADC. Conversion
33  * can be triggered by some as yet unspecified sources, including by
34  * software. Each trigger might launch a single scan or a continuous
35  * scan based on the configured adc mode. Single channel conversion is
36  * not presently supported.
37  *
38  * Each conversion result is stored temporarily in the adc peripheral's
39  * state variable, and a handler function too be provided by the
40  * application is called with the ADC interface number, channel number,
41  * and a pointer to the stored result. The application must provide this
42  * function to correctly dispatch the value to the actual consumer.
43  *
44  * This interface does not support at all the following features:
45  * - Analog Watchdog
46  * - Delayed Conversion Mode
47  * - Discontinuous Conversion Mode
48  * - Reduced Resolution
49  * - DMA integration
50  * - ADF integration
51  * - Bulb, Trigger, and Plus sampling modes
52  * - Offsets
53  *
54  * Some early steps are taken to implement the following features, but
55  * applications should treat them as unsupported:
56  * - Injected Groups (some hooks included)
57  * - Oversampling
58  *
59  * Most other options are static and are hard-coded or otherwise specified
60  * at compile time only. Enhancement of the interface to allow these
61  * missing features, variable sampling rate and sampling time, simultaneous
62  * sampling, DMA, etc. may be considered if serious use cases present
63  * themselves. For the moment, applications are expected to implement
64  * their own ADC interfaces as needed for complex use cases.
65  *
66  * This implementation also does not support arbitrary channel ordering.
67  * Specifically, we restrict the channel configuration to each channel
68  * appearing at most once in the the regular sequence register, and the
69  * conversion sequence to always be a monotonic increasing channel
70  * number.
71  *
72  * Implementation Notes
73  * --------------------
74  * Most ADC config changes can only be done when the ADC is disabled.
75  * This implementation is not currently designed to handle on-the-fly
76  * changes, and the init functions are generally expected to be called
77  * exactly once during system power up.
78  *
79  * ADC Modes
80  * ---------
81  *
82  * ADC_MODE_DISABLED:
83  * ADC remains off and unconfigured. If the mode is set to disabled,
84  * this interface will be unavailable to the application, even at
85  * runtime. However, the application can use a fully independent
86  * interface, provided by iteself, though which it which can configure
87  * and use the ADC with full flexibility.
88  *
89  * ADC_MODE_SINGLE:
90  * Not presently supported by this implementation. It might later be
91  * added by using instead a dynamic conversion group with a single
92  * channel. We'll probably need to couple it with some kind of mutex
93  * or so to make sure individual calls do not clobber each other.
94  *
95  * ADC_MODE_SCAN:
96  * Scan though all enabled channels once, stop when done.
97  *
98  * ADC_MODE_CONTINUOUS:
99  * Scan though all enabled channels once, repeat when done.
100  *
101  * This chip also supports dual ADC modes, which can be used for
102  * simultaneous sampling. While we do want to support this, we're
103  * deferring it for the moment.
104  */
